OOP (Object Oriented Programming) 4대 특징
	- 추상화 (Abstraction) : 구현 상세내용은 숨기는 것
	- 캡슐화 (Encapsulation) : 관련된 데이터와 그 데이터에 대한 연산을 하나로 그룹핑 하는 것.
	- 다향성 (Polymorphism) : 동일한 이름의 변수 또는 메소드가 여러 형태를 가지는 것.
	- 상속 (Inheritance) : 부모 또는 조상 클래스로 부터 프로퍼티와 메소드를 상속 받는 것.

정보은닉 : 접근제어자(Access Modifier : public, protected, private)를 통해 정보를 숨기는 것.

OOP 언어를 통해 소프트웨어를 개발 특징
	- 객체들을 생성하고 그 객체들 간의 상호작용을 구현하는 것.
	- 객체들을 생성하기 위해서는 설계도가 필요 : 클래스

클래스 생성 방법
	- 소프트웨어 개발을 의뢰한 고객과의 미팅을 통한 논의 필요.
	- 요구사항을 기반으로 설계, 그 기본 중 하나가 클래스.

UML (Unified Modeling Language) 

자바의 자료유형(data type)
기초형 : byte, char, short, int ,long, float, double, boolean
참조형 : String, 배열(int[], char[],...), 클래스(Date, Scanner, Random...), 사용자가 만든 클래스(User Defined Data Type)
